HTML#

Este tutorial irá guiá-lo na criação de uma aplicação HTML SSR (Server-Side Rendering) com Gez a partir do zero. Através de um exemplo completo, mostraremos como usar o framework Gez para criar uma aplicação com renderização no servidor.

Estrutura do Projeto#

Primeiro, vamos entender a estrutura básica do projeto:

1.
2├── package.json         # Arquivo de configuração do projeto, define dependências e scripts
3├── tsconfig.json        # Arquivo de configuração do TypeScript, define opções de compilação
4└── src                  # Diretório de código-fonte
5    ├── app.ts           # Componente principal da aplicação, define a estrutura e lógica da página
6    ├── create-app.ts    # Fábrica de criação de instância da aplicação, responsável pela inicialização
7    ├── entry.client.ts  # Arquivo de entrada do cliente, lida com a renderização no navegador
8    ├── entry.node.ts    # Arquivo de entrada do servidor Node.js, configura o ambiente de desenvolvimento e inicia o servidor
9    └── entry.server.ts  # Arquivo de entrada do servidor, lida com a lógica de renderização SSR

Configuração do Projeto#

package.json#

Crie o arquivo package.json para configurar as dependências e scripts do projeto:

package.json
1{
2  "name": "ssr-demo-html",
3  "version": "1.0.0",
4  "type": "module",
5  "private": true,
6  "scripts": {
7    "dev": "gez dev",
8    "build": "npm run build:dts && npm run build:ssr",
9    "build:ssr": "gez build",
10    "preview": "gez preview",
11    "start": "NODE_ENV=production node dist/index.js",
12    "build:dts": "tsc --declaration --emitDeclarationOnly --outDir dist/src"
13  },
14  "dependencies": {
15    "@gez/core": "*"
16  },
17  "devDependencies": {
18    "@gez/rspack": "*",
19    "@types/node": "22.8.6",
20    "typescript": "^5.7.3"
21  }
22}

Após criar o arquivo package.json, instale as dependências do projeto. Você pode usar um dos seguintes comandos:

1pnpm install
2# ou
3yarn install
4# ou
5npm install

Isso instalará todos os pacotes necessários, incluindo TypeScript e dependências relacionadas ao SSR.

Estrutura do Código-Fonte#

app.ts#

Crie o componente principal da aplicação em src/app.ts, implementando a estrutura e lógica da página:

src/app.ts
1/**
2 * @file Componente de Exemplo
3 * @description Exibe um título de página com atualização automática de tempo, demonstrando funcionalidades básicas do Gez
4 */
5
6export default class App {
7    /**
8     * Hora atual, no formato ISO
9     * @type {string}
10     */
11    public time = '';
12
13    /**
14     * Cria uma instância da aplicação
15     * @param {SsrContext} [ssrContext] - Contexto do servidor, contendo metadados de importação
16     */
17    public constructor(public ssrContext?: SsrContext) {
18        // Nenhuma inicialização adicional necessária no construtor
19    }
20
21    /**
22     * Renderiza o conteúdo da página
23     * @returns {string} Retorna a estrutura HTML da página
24     */
25    public render(): string {
26        // Garante a coleta correta de metadados de importação no ambiente do servidor
27        if (this.ssrContext) {
28            this.ssrContext.importMetaSet.add(import.meta);
29        }
30
31        return `
32        <div id="app">
33            <h1><a href="https://www.jsesm.com/guide/frameworks/html.html" target="_blank">Início Rápido com Gez</a></h1>
34            <time datetime="${this.time}">${this.time}</time>
35        </div>
36        `;
37    }
38
39    /**
40     * Inicialização no cliente
41     * @throws {Error} Lança um erro se o elemento de exibição de tempo não for encontrado
42     */
43    public onClient(): void {
44        // Obtém o elemento de exibição de tempo
45        const time = document.querySelector('#app time');
46        if (!time) {
47            throw new Error('Elemento de exibição de tempo não encontrado');
48        }
49
50        // Configura um temporizador para atualizar o tempo a cada segundo
51        setInterval(() => {
52            this.time = new Date().toISOString();
53            time.setAttribute('datetime', this.time);
54            time.textContent = this.time;
55        }, 1000);
56    }
57
58    /**
59     * Inicialização no servidor
60     */
61    public onServer(): void {
62        this.time = new Date().toISOString();
63    }
64}
65
66/**
67 * Interface de contexto do servidor
68 * @interface
69 */
70export interface SsrContext {
71    /**
72     * Conjunto de metadados de importação
73     * @type {Set<ImportMeta>}
74     */
75    importMetaSet: Set<ImportMeta>;
76}

entry.node.ts#

Crie o arquivo entry.node.ts para configurar o ambiente de desenvolvimento e iniciar o servidor:

src/entry.node.ts
1/**
2 * @file Arquivo de Entrada do Servidor Node.js
3 * @description Responsável pela configuração do ambiente de desenvolvimento e inicialização do servidor, fornecendo o ambiente de execução SSR
4 */
5
6import http from 'node:http';
7import type { GezOptions } from '@gez/core';
8
9export default {
10    /**
11     * Configura o criador de aplicação para o ambiente de desenvolvimento
12     * @description Cria e configura a instância da aplicação Rspack para construção e atualização em tempo real no ambiente de desenvolvimento
13     * @param gez Instância do Gez, fornecendo funcionalidades principais e interfaces de configuração
14     * @returns Retorna a instância configurada da aplicação Rspack, suportando HMR e visualização em tempo real
15     */
16    async devApp(gez) {
17        return import('@gez/rspack').then((m) =>
18            m.createRspackHtmlApp(gez, {
19                config(context) {
20                    // Personalize a configuração de compilação do Rspack aqui
21                }
22            })
23        );
24    },
25
26    /**
27     * Configura e inicia o servidor HTTP
28     * @description Cria uma instância do servidor HTTP, integra middleware do Gez e processa requisições SSR
29     * @param gez Instância do Gez, fornecendo middleware e funcionalidades de renderização
30     */
31    async server(gez) {
32        const server = http.createServer((req, res) => {
33            // Usa middleware do Gez para processar a requisição
34            gez.middleware(req, res, async () => {
35                // Executa a renderização no servidor
36                const rc = await gez.render({
37                    params: { url: req.url }
38                });
39                res.end(rc.html);
40            });
41        });
42
43        server.listen(3000, () => {
44            console.log('Servidor iniciado: http://localhost:3000');
45        });
46    }
47} satisfies GezOptions;

Este arquivo é o ponto de entrada para a configuração do ambiente de desenvolvimento e inicialização do servidor, contendo duas funcionalidades principais:

  1. Função devApp: Responsável por criar e configurar a instância da aplicação Rspack para o ambiente de desenvolvimento, suportando atualização em tempo real e visualização.
  2. Função server: Responsável por criar e configurar o servidor HTTP, integrando middleware do Gez para processar requisições SSR.

entry.server.ts#

Crie o arquivo de entrada para renderização no servidor src/entry.server.ts:

src/entry.server.ts
1/**
2 * @file Arquivo de Entrada para Renderização no Servidor
3 * @description Responsável pelo fluxo de renderização no servidor, geração de HTML e injeção de recursos
4 */
5
6import type { RenderContext } from '@gez/core';
7import type App from './app';
8import type { SsrContext } from './app';
9import { createApp } from './create-app';
10
11// Encapsula a lógica de geração de conteúdo da página
12const renderToString = (app: App, ssrContext: SsrContext): string => {
13    // Injeta o contexto de renderização no servidor na instância da aplicação
14    app.ssrContext = ssrContext;
15    // Inicializa o servidor
16    app.onServer();
17
18    // Gera o conteúdo da página
19    return app.render();
20};
21
22export default async (rc: RenderContext) => {
23    // Cria a instância da aplicação, retornando um objeto contendo a instância app
24    const { app } = createApp();
25    // Usa renderToString para gerar o conteúdo da página
26    const html = renderToString(app, {
27        importMetaSet: rc.importMetaSet
28    });
29
30    // Submete a coleta de dependências, garantindo que todos os recursos necessários sejam carregados
31    await rc.commit();
32
33    // Gera a estrutura HTML completa
34    rc.html = `<!DOCTYPE html>
35<html lang="pt-BR">
36<head>
37    ${rc.preload()}
38    <title>Início Rápido com Gez</title>
39    ${rc.css()}
40</head>
41<body>
42    ${html}
43    ${rc.importmap()}
44    ${rc.moduleEntry()}
45    ${rc.modulePreload()}
46</body>
47</html>
48`;
49};

Executando o Projeto#

Após configurar os arquivos acima, você pode executar o projeto com os seguintes comandos:

  1. Modo de desenvolvimento:
1npm run dev
  1. Construir o projeto:
1npm run build
  1. Executar em produção:
1npm run start

Agora, você criou com sucesso uma aplicação HTML SSR com Gez! Acesse http://localhost:3000 para ver o resultado.
