プロジェクトの初期化#

新しい Gez プロジェクトを作成して初期化するには、以下の手順に従ってください：

1mkdir your-app-name
2cd your-app-name

フレームワークバージョンの選択#

Gez フレームワークは、さまざまな技術スタックのバージョンを提供しており、それぞれが異なるアプリケーションシナリオに最適化されています。プロジェクトの要件に応じて適切なバージョンを選択してください：

HTML#

極限までシンプルなアーキテクチャを求めるプロジェクトに適しています：

  • 外部依存関係なし、すぐに使用可能
  • ネイティブ JavaScript 開発体験
  • 静的サイトや軽量アプリケーションの構築に適しています
  • プログレッシブな機能拡張をサポート

HTML バージョンの詳細ドキュメントを確認する

Vue2#

エンタープライズレベルのアプリケーション開発に適しています：

  • 完全な TypeScript サポート
  • 豊富なサードパーティコンポーネントエコシステム
  • 成熟した開発ツールチェーン
  • 本番環境での検証済みの安定性

Vue2 バージョンの詳細ドキュメントを確認する

Vue3#

モダンな Web アプリケーション開発に適しています：

  • Proxy ベースのリアクティブシステム
  • コンポジション API のサポート
  • より優れたランタイムパフォーマンス
  • より小さなバンドルサイズ

Vue3 バージョンの詳細ドキュメントを確認する

Preact+HTM#

軽量で高性能を求めるプロジェクトに適しています：

  • 極小のランタイムサイズ（3KB）
  • ネイティブ JavaScript テンプレート構文
  • React 互換の API
  • 優れたパフォーマンス

Preact+HTM バージョンの詳細ドキュメントを確認する

AI 支援開発#

Gez フレームワークは、開発効率を大幅に向上させる AI 支援開発機能を提供しています：

  1. 適切なフレームワークバージョンのドキュメントを選択
  2. ドキュメントの内容を AI アシスタントに提供
  3. AI が自動的にプロジェクトの骨組みと設定ファイルを生成
ヒント

AI 支援開発は、プロジェクトの初期化速度を向上させるだけでなく、プロジェクト構造がベストプラクティスに準拠していることを保証します。
