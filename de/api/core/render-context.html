RenderContext#

RenderContext ist die Kernklasse im Gez-Framework und verantwortlich für die Verwaltung des gesamten Lebenszyklus des serverseitigen Renderings (SSR). Sie bietet eine umfassende API zur Handhabung von Rendering-Kontexten, Ressourcenverwaltung, Zustandssynchronisierung und anderen wichtigen Aufgaben:

  • Rendering-Steuerung: Verwaltung des serverseitigen Rendering-Prozesses, Unterstützung für Mehrfach-Eintrags-Rendering, bedingtes Rendering usw.
  • Ressourcenverwaltung: Intelligente Sammlung und Einbindung von statischen Ressourcen wie JS, CSS usw. zur Optimierung der Ladeleistung
  • Zustandssynchronisierung: Handhabung der serverseitigen Zustandsserialisierung, um eine korrekte Client-Aktivierung (Hydration) sicherzustellen
  • Routing-Kontrolle: Unterstützung für serverseitige Umleitungen, Statuscode-Einstellungen und andere erweiterte Funktionen

Typdefinitionen#

ServerRenderHandle#

Typdefinition für die serverseitige Rendering-Handler-Funktion.

1type ServerRenderHandle = (rc: RenderContext) => Promise<void> | void;

Die serverseitige Rendering-Handler-Funktion ist eine asynchrone oder synchrone Funktion, die eine RenderContext-Instanz als Parameter empfängt und zur Verarbeitung der serverseitigen Rendering-Logik verwendet wird.

entry.node.ts
1// 1. Asynchrone Handler-Funktion
2export default async (rc: RenderContext) => {
3  const app = createApp();
4  const html = await renderToString(app);
5  rc.html = html;
6};
7
8// 2. Synchrone Handler-Funktion
9export const simple = (rc: RenderContext) => {
10  rc.html = '<h1>Hello World</h1>';
11};

RenderFiles#

Typdefinition für die Liste der während des Renderings gesammelten Ressourcendateien.

1interface RenderFiles {
2  js: string[];
3  css: string[];

5  resources: string[];
6}
  • js: Liste der JavaScript-Dateien
  • css: Liste der Stylesheet-Dateien
  • modulepreload: Liste der ESM-Module, die vorab geladen werden müssen
  • resources: Liste anderer Ressourcendateien (Bilder, Schriftarten usw.)
1// Beispiel für eine Ressourcendateiliste
2rc.files = {
3  js: [
4    '/assets/entry-client.js',
5    '/assets/vendor.js'
6  ],
7  css: [
8    '/assets/main.css',
9    '/assets/vendor.css'
10  ],
11  modulepreload: [
12    '/assets/Home.js',
13    '/assets/About.js'
14  ],
15  resources: [
16    '/assets/logo.png',
17    '/assets/font.woff2'
18  ]
19};

ImportmapMode#

Definiert den Generierungsmodus für Importmaps.

1type ImportmapMode = 'inline' | 'js';
  • inline: Der Importmap-Inhalt wird direkt in das HTML eingebettet, geeignet für folgende Szenarien:
    • Reduzierung der Anzahl der HTTP-Anfragen
    • Der Importmap-Inhalt ist klein
    • Hohe Anforderungen an die Ladeleistung der ersten Seite
  • js: Der Importmap-Inhalt wird als separate JS-Datei generiert, geeignet für folgende Szenarien:
    • Der Importmap-Inhalt ist groß
    • Nutzung des Browser-Caching-Mechanismus
    • Mehrere Seiten teilen sich dieselbe Importmap

Rendering-Kontextklasse, verantwortlich für die Ressourcenverwaltung und HTML-Generierung während des serverseitigen Renderings (SSR).

Instanzoptionen#

Definiert die Konfigurationsoptionen für den Rendering-Kontext.

1interface RenderContextOptions {
2  base?: string
3  entryName?: string
4  params?: Record<string, any>
5  importmapMode?: ImportmapMode
6}

base#

  • Typ: string
  • Standardwert: ''

Der Basispfad für statische Ressourcen.

  • Alle statischen Ressourcen (JS, CSS, Bilder usw.) werden relativ zu diesem Pfad geladen
  • Unterstützt dynamische Konfiguration zur Laufzeit, ohne Neubuild erforderlich
  • Häufig verwendet in mehrsprachigen Websites, Micro-Frontend-Anwendungen usw.

entryName#

  • Typ: string
  • Standardwert: 'default'

Der Name der serverseitigen Rendering-Eintrittsfunktion. Wird verwendet, um die zu verwendende Rendering-Funktion anzugeben, wenn ein Modul mehrere Rendering-Funktionen exportiert.

src/entry.server.ts
1export const mobile = async (rc: RenderContext) => {
2  // Rendering-Logik für Mobilgeräte
3};
4
5export const desktop = async (rc: RenderContext) => {
6  // Rendering-Logik für Desktop-Geräte
7};

params#

  • Typ: Record<string, any>
  • Standardwert: {}

Rendering-Parameter. Kann beliebige Parameter an die Rendering-Funktion übergeben, häufig verwendet für die Übergabe von Anfrageinformationen (URL, Query-Parameter usw.).

1const rc = await gez.render({
2  params: {
3    url: req.url,
4    lang: 'zh-CN',
5    theme: 'dark'
6  }
7});

importmapMode#

  • Typ: 'inline' | 'js'
  • Standardwert: 'inline'

Generierungsmodus für Import Maps:

  • inline: Der Importmap-Inhalt wird direkt in das HTML eingebettet

  • js: Der Importmap-Inhalt wird als separate JS-Datei generiert

Instanzeigenschaften#

gez#

  • Typ: Gez
  • Nur Lesen: true

Referenz auf die Gez-Instanz. Wird verwendet, um auf Kernfunktionen und Konfigurationsinformationen des Frameworks zuzugreifen.

redirect#

  • Typ: string | null
  • Standardwert: null

Umleitungsadresse. Nach der Festlegung kann der Server basierend auf diesem Wert eine HTTP-Umleitung durchführen, häufig verwendet für Login-Validierung, Berechtigungskontrolle usw.

entry.node.ts
1// Beispiel für Login-Validierung
2export default async (rc: RenderContext) => {
3  if (!isLoggedIn()) {
4    rc.redirect = '/login';
5    rc.status = 302;
6    return;
7  }
8  // Weiteres Rendering der Seite...
9};
10
11// Beispiel für Berechtigungskontrolle
12export default async (rc: RenderContext) => {
13  if (!hasPermission()) {
14    rc.redirect = '/403';
15    rc.status = 403;
16    return;
17  }
18  // Weiteres Rendering der Seite...
19};

status#

  • Typ: number | null
  • Standardwert: null

HTTP-Antwortstatuscode. Kann beliebige gültige HTTP-Statuscodes festlegen, häufig verwendet für Fehlerbehandlung, Umleitungen usw.

entry.node.ts
1// Beispiel für 404-Fehlerbehandlung
2export default async (rc: RenderContext) => {
3  const page = await findPage(rc.params.url);
4  if (!page) {
5    rc.status = 404;
6    // Rendering der 404-Seite...
7    return;
8  }
9  // Weiteres Rendering der Seite...
10};
11
12// Beispiel für temporäre Umleitung
13export default async (rc: RenderContext) => {
14  if (needMaintenance()) {
15    rc.redirect = '/maintenance';
16    rc.status = 307; // Temporäre Umleitung, behält die Anfragemethode bei
17    return;
18  }
19  // Weiteres Rendering der Seite...
20};

html#

  • Typ: string
  • Standardwert: ''

HTML-Inhalt. Wird verwendet, um den final generierten HTML-Inhalt festzulegen und abzurufen, wobei Platzhalter für den Basispfad automatisch verarbeitet werden.

entry.node.ts
1// Grundlegende Verwendung
2export default async (rc: RenderContext) => {
3  // Festlegen des HTML-Inhalts
4  rc.html = `
5    <!DOCTYPE html>
6    <html>
7      <head>
8        ${rc.preload()}
9        ${rc.css()}
10      </head>
11      <body>
12        <div id="app">Hello World</div>
13        ${rc.importmap()}
14        ${rc.moduleEntry()}
15        ${rc.modulePreload()}
16      </body>
17    </html>
18  `;
19};
20
21// Dynamischer Basispfad
22const rc = await gez.render({
23  base: '/app',  // Festlegen des Basispfads
24  params: { url: req.url }
25});
26
27// Platzhalter im HTML werden automatisch ersetzt:
28// [[[___GEZ_DYNAMIC_BASE___]]]/your-app-name/css/style.css
29// Wird ersetzt durch:
30// /app/your-app-name/css/style.css

base#

  • Typ: string
  • Nur Lesen: true
  • Standardwert: ''

Der Basispfad für statische Ressourcen. Alle statischen Ressourcen (JS, CSS, Bilder usw.) werden relativ zu diesem Pfad geladen, unterstützt dynamische Konfiguration zur Laufzeit.

1// Grundlegende Verwendung
2const rc = await gez.render({
3  base: '/gez',  // Festlegen des Basispfads
4  params: { url: req.url }
5});
6
7// Beispiel für mehrsprachige Websites
8const rc = await gez.render({
9  base: '/cn',  // Chinesische Website
10  params: { lang: 'zh-CN' }
11});
12
13// Beispiel für Micro-Frontend-Anwendungen
14const rc = await gez.render({
15  base: '/app1',  // Unteranwendung 1
16  params: { appId: 1 }
17});

entryName#

  • Typ: string
  • Nur Lesen: true
  • Standardwert: 'default'

Der Name der serverseitigen Rendering-Eintrittsfunktion. Wird verwendet, um die zu verwendende Rendering-Funktion aus entry.server.ts auszuwählen.

entry.node.ts
1// Standard-Eintrittsfunktion
2export default async (rc: RenderContext) => {
3  // Standard-Rendering-Logik
4};
5
6// Mehrere Eintrittsfunktionen
7export const mobile = async (rc: RenderContext) => {
8  // Rendering-Logik für Mobilgeräte
9};
10
11export const desktop = async (rc: RenderContext) => {
12  // Rendering-Logik für Desktop-Geräte
13};
14
15// Auswahl der Eintrittsfunktion basierend auf dem Gerätetyp
16const rc = await gez.render({
17  entryName: isMobile ? 'mobile' : 'desktop',
18  params: { url: req.url }
19});

params#

  • Typ: Record<string, any>
  • Nur Lesen: true
  • Standardwert: {}

Rendering-Parameter. Kann während des serverseitigen Renderings übergeben und abgerufen werden, häufig verwendet für die Übergabe von Anfrageinformationen, Seitenkonfigurationen usw.

1// Grundlegende Verwendung - Übergabe von URL und Spracheinstellungen
2const rc = await gez.render({
3  params: {
4    url: req.url,
5    lang: 'zh-CN'
6  }
7});
8
9// Seitenkonfiguration - Festlegen von Thema und Layout
10const rc = await gez.render({
11  params: {
12    theme: 'dark',
13    layout: 'sidebar'
14  }
15});
16
17// Umgebungskonfiguration - Einbindung der API-Adresse
18const rc = await gez.render({
19  params: {
20    apiBaseUrl: process.env.API_BASE_URL,
21    version: '1.0.0'
22  }
23});

importMetaSet#

  • Typ: Set<ImportMeta>

Sammlung von Modulabhängigkeiten. Während des Komponenten-Renderings werden automatisch Modulabhängigkeiten verfolgt und aufgezeichnet, wobei nur die tatsächlich verwendeten Ressourcen gesammelt werden.

1// Grundlegende Verwendung
2const renderToString = (app: any, context: { importMetaSet: Set<ImportMeta> }) => {
3  // Automatische Sammlung von Modulabhängigkeiten während des Renderings
4  // Das Framework ruft automatisch context.importMetaSet.add(import.meta) während des Komponenten-Renderings auf
5  // Entwickler müssen die Abhängigkeitssammlung nicht manuell behandeln
6  return '<div id="app">Hello World</div>';
7};
8
9// Verwendungsbeispiel
10const app = createApp();
11const html = await renderToString(app, {
12  importMetaSet: rc.importMetaSet
13});

files#

  • Typ: RenderFiles

Liste der Ressourcendateien:

  • js: Liste der JavaScript-Dateien
  • css: Liste der Stylesheet-Dateien
  • modulepreload: Liste der ESM-Module, die vorab geladen werden müssen
  • resources: Liste anderer Ressourcendateien (Bilder, Schriftarten usw.)
1// Ressourcensammlung
2await rc.commit();
3
4// Ressourceneinbindung
5rc.html = `
6  <!DOCTYPE html>
7  <html>
8  <head>
9    <!-- Vorab geladene Ressourcen -->
10    ${rc.preload()}
11    <!-- Einbindung von Stylesheets -->
12    ${rc.css()}
13  </head>
14  <body>
15    ${html}
16    ${rc.importmap()}
17    ${rc.moduleEntry()}
18    ${rc.modulePreload()}
19  </body>
20  </html>
21`;

  • Standardwert: 'inline'

  • inline: Der Importmap-Inhalt wird direkt in das HTML eingebettet

  • js: Der Importmap-Inhalt wird als separate JS-Datei generiert

Instanzmethoden#

serialize()#

  • Parameter:
    • input: any - Die zu serialisierenden Daten
    • options?: serialize.SerializeJSOptions - Serialisierungsoptionen
  • Rückgabewert: string

Serialisiert ein JavaScript-Objekt zu einem String. Wird verwendet, um Zustandsdaten während des serverseitigen Renderings zu serialisieren und sicher in das HTML einzubetten.

1const state = {
2  user: { id: 1, name: 'Alice' },
3  timestamp: new Date()
4};
5
6rc.html = `
7  <script>
8    window.__INITIAL_STATE__ = ${rc.serialize(state)};
9  </script>
10`;

state()#

  • Parameter:
    • varName: string - Variablenname
    • data: Record<string, any> - Zustandsdaten
  • Rückgabewert: string

Serialisiert Zustandsdaten und bindet sie in das HTML ein. Verwendet sichere Serialisierungsmethoden für die Datenverarbeitung und unterstützt komplexe Datenstrukturen.

1const userInfo = {
2  id: 1,
3  name: 'John',
4  roles: ['admin']
5};
6
7rc.html = `
8  <head>
9    ${rc.state('__USER__', userInfo)}
10  </head>
11`;

commit()#

  • Rückgabewert: Promise<void>

Übergibt die Abhängigkeitssammlung und aktualisiert die Ressourcenliste. Sammelt alle verwendeten Module aus importMetaSet und analysiert die spezifischen Ressourcen jedes Moduls basierend auf der Manifest-Datei.

1// Rendering und Übergabe der Abhängigkeiten
2const html = await renderToString(app, {
3  importMetaSet: rc.importMetaSet
4});
5
6// Übergabe der Abhängigkeitssammlung
7await rc.commit();

preload()#

  • Rückgabewert: string

Generiert Ressourcen-Vorlade-Tags. Wird verwendet, um CSS- und JavaScript-Ressourcen vorab zu laden, unterstützt Prioritätskonfiguration und verarbeitet automatisch den Basispfad.

1rc.html = `
2  <!DOCTYPE html>
3  <html>
4  <head>
5    ${rc.preload()}
6    ${rc.css()}  <!-- Einbindung von Stylesheets -->
7  </head>
8  <body>
9    ${html}
10    ${rc.importmap()}
11    ${rc.moduleEntry()}
12    ${rc.modulePreload()}
13  </body>
14  </html>
15`;

css()#

  • Rückgabewert: string

Generiert CSS-Stylesheet-Tags. Bindet die gesammelten CSS-Dateien ein und stellt sicher, dass die Stylesheets in der richtigen Reihenfolge geladen werden.

1rc.html = `
2  <head>
3    ${rc.css()}  <!-- Einbindung aller gesammelten Stylesheets -->
4  </head>
5`;

2  <head>
3    ${rc.importmap()}  <!-- Einbindung der Import-Map -->
