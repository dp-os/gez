模組連結#

Gez 框架提供了一套完整的模組連結機制，用於管理服務間的程式碼共享和依賴關係。該機制基於 ESM（ECMAScript Module）規範實現，支援原始碼層級的模組匯出和匯入，以及完整的依賴管理功能。

核心概念#

模組匯出#

模組匯出是將服務中的特定程式碼單元（如元件、工具函數等）以 ESM 格式對外暴露的過程。支援兩種匯出類型：

  • 原始碼匯出：直接匯出專案中的原始碼檔案
  • 依賴匯出：匯出專案使用的第三方依賴套件

模組匯入#

模組匯入是在服務中引用其他服務匯出的程式碼單元的過程。支援多種安裝方式：

  • 原始碼安裝：適用於開發環境，支援即時修改和熱更新
  • 軟體包安裝：適用於生產環境，直接使用建置產物

預載機制#

為了優化服務效能，Gez 實現了智能的模組預載機制：

  1. 依賴分析

    • 建置時分析元件間的依賴關係
    • 識別關鍵路徑上的核心模組
    • 確定模組的載入優先級

  2. 載入策略

    • 立即載入：關鍵路徑上的核心模組
    • 延遲載入：非關鍵功能模組
    • 按需載入：條件渲染的模組

  3. 資源優化

    • 智能的程式碼分割策略
    • 模組層級的快取管理
    • 按需編譯和打包

模組匯出#

配置說明#

在 entry.node.ts 中配置需要匯出的模組：

src/entry.node.ts
1import type { GezOptions } from '@gez/core';
2
3export default {
4    modules: {
5        exports: [
6            // 匯出原始碼檔案
7            'root:src/components/button.vue',  // Vue 元件
8            'root:src/utils/format.ts',        // 工具函數
9            // 匯出第三方依賴
10            'npm:vue',                         // Vue 框架
11            'npm:vue-router'                   // Vue Router
12        ]
13    }
14} satisfies GezOptions;

匯出配置支援兩種類型：

  • root:*：匯出原始碼檔案，路徑相對於專案根目錄
  • npm:*：匯出第三方依賴，直接指定套件名稱

模組匯入#

配置說明#

在 entry.node.ts 中配置需要匯入的模組：

src/entry.node.ts
1import type { GezOptions } from '@gez/core';
2
3export default {
4    modules: {
5        // 匯入配置
6        imports: {
7            // 原始碼安裝：指向建置產物目錄
8            'ssr-remote': 'root:./node_modules/ssr-remote/dist',
9            // 軟體包安裝：指向套件目錄
10            'other-remote': 'root:./node_modules/other-remote'
11        },
12        // 外部依賴配置
13        externals: {
14            // 使用遠端模組中的依賴
15            'vue': 'ssr-remote/npm/vue',
16            'vue-router': 'ssr-remote/npm/vue-router'
17        }
18    }
19} satisfies GezOptions;

配置項說明：

  1. imports：配置遠端模組的本地路徑

    • 原始碼安裝：指向建置產物目錄（dist）
    • 軟體包安裝：直接指向套件目錄

  2. externals：配置外部依賴

    • 用於共享遠端模組中的依賴
    • 避免重複打包相同依賴
    • 支援多個模組共享依賴

安裝方式#

原始碼安裝#

適用於開發環境，支援即時修改和熱更新。

  1. Workspace 方式 推薦在 Monorepo 專案中使用：
package.json
1{
2    "devDependencies": {
3        "ssr-remote": "workspace:*"
4    }
5}
  1. Link 方式 用於本地開發除錯：
package.json
1{
2    "devDependencies": {
3        "ssr-remote": "link:../ssr-remote"
4    }
5}

軟體包安裝#

適用於生產環境，直接使用建置產物。

  1. NPM Registry 透過 npm registry 安裝：
package.json
1{
2    "dependencies": {
3        "ssr-remote": "^1.0.0"
4    }
5}
  1. 靜態伺服器 透過 HTTP/HTTPS 協定安裝：
package.json
1{
2    "dependencies": {
3        "ssr-remote": "https://cdn.example.com/ssr-remote/1.0.0.tgz"
4    }
5}

軟體包建置#

配置說明#

在 entry.node.ts 中配置建置選項：

src/entry.node.ts
1import type { GezOptions } from '@gez/core';
2
3export default {
4    // 模組匯出配置
5    modules: {
6        exports: [
7            'root:src/components/button.vue',
8            'root:src/utils/format.ts',
9            'npm:vue'
10        ]
11    },
12    // 建置配置
13    pack: {
14        // 啟用建置
15        enable: true,
16
17        // 輸出配置
18        outputs: [
19            'dist/client/versions/latest.tgz',
20            'dist/client/versions/1.0.0.tgz'
21        ],
22
23        // 自訂 package.json
24        packageJson: async (gez, pkg) => {
25            pkg.version = '1.0.0';
26            return pkg;
27        },
28
29        // 建置前處理
30        onBefore: async (gez, pkg) => {
31            // 產生類型宣告
32            // 執行測試用例
33            // 更新文件等
34        },
35
36        // 建置後處理
37        onAfter: async (gez, pkg, file) => {
38            // 上傳到 CDN
39            // 發布到 npm 倉庫
40            // 部署到測試環境等
41        }
42    }
43} satisfies GezOptions;

建置產物#

your-app-name.tgz ├── package.json # 套件資訊 ├── index.js # 生產環境入口 ├── server/ # 伺服器端資源 │ └── manifest.json # 伺服器端資源映射 ├── node/ # Node.js 執行環境 └── client/ # 客戶端資源 └── manifest.json # 客戶端資源映射

發布流程#

1# 1. 建置生產版本
2gez build
3
4# 2. 發布到 npm
5npm publish dist/versions/your-app-name.tgz

開發環境配置#

  • 依賴管理
