PackConfig#

PackConfig सॉफ़्टवेयर पैकेज पैकेजिंग कॉन्फ़िगरेशन इंटरफ़ेस है, जिसका उपयोग सेवा के बिल्ड आउटपुट को मानक npm .tgz प्रारूप में पैकेज करने के लिए किया जाता है।

  • मानकीकरण: npm मानक .tgz पैकेजिंग प्रारूप का उपयोग करता है
  • पूर्णता: मॉड्यूल के स्रोत कोड, टाइप डिक्लेरेशन और कॉन्फ़िगरेशन फ़ाइलों सहित सभी आवश्यक फ़ाइलें शामिल हैं
  • संगतता: npm इकोसिस्टम के साथ पूरी तरह से संगत, मानक पैकेज प्रबंधन वर्कफ़्लो का समर्थन करता है

प्रकार परिभाषा#

1interface PackConfig {
2    enable?: boolean;
3    outputs?: string | string[] | boolean;
4    packageJson?: (gez: Gez, pkg: Record<string, any>) => Promise<Record<string, any>>;
5    onBefore?: (gez: Gez, pkg: Record<string, any>) => Promise<void>;
6    onAfter?: (gez: Gez, pkg: Record<string, any>, file: Buffer) => Promise<void>;
7}

PackConfig#

enable#

पैकेजिंग सुविधा को सक्षम करें या नहीं। सक्षम करने पर बिल्ड आउटपुट को मानक npm .tgz प्रारूप में पैकेज किया जाएगा।

  • प्रकार: boolean
  • डिफ़ॉल्ट मान: false

outputs#

आउटपुट पैकेज फ़ाइल पथ निर्दिष्ट करें। निम्नलिखित कॉन्फ़िगरेशन विधियों का समर्थन करता है:

  • string: एकल आउटपुट पथ, जैसे 'dist/versions/my-app.tgz'
  • string[]: एकाधिक आउटपुट पथ, एक साथ कई संस्करण उत्पन्न करने के लिए
  • boolean: true होने पर डिफ़ॉल्ट पथ 'dist/client/versions/latest.tgz' का उपयोग करता है

packageJson#

package.json सामग्री को अनुकूलित करने के लिए कॉलबैक फ़ंक्शन। पैकेजिंग से पहले कॉल किया जाता है, package.json की सामग्री को अनुकूलित करने के लिए उपयोग किया जाता है।

  • पैरामीटर:
    • gez: Gez - Gez उदाहरण
    • pkg: any - मूल package.json सामग्री
  • रिटर्न मान: Promise<any> - संशोधित package.json सामग्री

सामान्य उपयोग:

  • पैकेज नाम और संस्करण संख्या संशोधित करना
  • निर्भरताएँ जोड़ना या अद्यतन करना
  • कस्टम फ़ील्ड जोड़ना
  • प्रकाशन संबंधी जानकारी कॉन्फ़िगर करना

उदाहरण:

1packageJson: async (gez, pkg) => {
2  // पैकेज जानकारी सेट करें
3  pkg.name = 'my-app';
4  pkg.version = '1.0.0';
5  pkg.description = 'मेरा एप्लिकेशन';
6
7  // निर्भरताएँ जोड़ें
8  pkg.dependencies = {
9    'vue': '^3.0.0',
10    'express': '^4.17.1'
11  };
12
13  // प्रकाशन कॉन्फ़िगरेशन जोड़ें
14  pkg.publishConfig = {
15    registry: 'https://registry.example.com'
16  };
17
18  return pkg;
19}

onBefore#

पैकेजिंग से पहले की तैयारी के लिए कॉलबैक फ़ंक्शन।

  • पैरामीटर:
    • gez: Gez - Gez उदाहरण
    • pkg: Record<string, any> - package.json सामग्री
  • रिटर्न मान: Promise<void>

सामान्य उपयोग:

  • अतिरिक्त फ़ाइलें जोड़ना (README, LICENSE आदि)
  • परीक्षण या बिल्ड सत्यापन निष्पादित करना
  • दस्तावेज़ या मेटाडेटा उत्पन्न करना
  • अस्थायी फ़ाइलों को साफ़ करना

उदाहरण:

1onBefore: async (gez, pkg) => {
2  // दस्तावेज़ जोड़ें
3  await fs.writeFile('dist/README.md', '# My App');
4  await fs.writeFile('dist/LICENSE', 'MIT License');
5
6  // परीक्षण निष्पादित करें
7  await runTests();
8
9  // दस्तावेज़ उत्पन्न करें
10  await generateDocs();
11
12  // अस्थायी फ़ाइलों को साफ़ करें
13  await cleanupTempFiles();
14}

onAfter#

पैकेजिंग पूर्ण होने के बाद के प्रसंस्करण के लिए कॉलबैक फ़ंक्शन। .tgz फ़ाइल उत्पन्न होने के बाद कॉल किया जाता है, पैकेज आउटपुट को संसाधित करने के लिए उपयोग किया जाता है।

  • पैरामीटर:
    • gez: Gez - Gez उदाहरण
    • pkg: Record<string, any> - package.json सामग्री
    • file: Buffer - पैकेज की गई फ़ाइल सामग्री
  • रिटर्न मान: Promise<void>

सामान्य उपयोग:

  • npm रिपॉजिटरी (सार्वजनिक या निजी) पर प्रकाशित करना
  • स्थिर संसाधन सर्वर पर अपलोड करना
  • संस्करण प्रबंधन निष्पादित करना
  • CI/CD प्रक्रिया ट्रिगर करना

उदाहरण:

1onAfter: async (gez, pkg, file) => {
2  // npm निजी रिपॉजिटरी पर प्रकाशित करें
3  await publishToRegistry(file, {
4    registry: 'https://registry.example.com'
5  });
6
7  // स्थिर संसाधन सर्वर पर अपलोड करें
8  await uploadToServer(file, 'https://assets.example.com/packages');
9
10  // संस्करण टैग बनाएँ
11  await createGitTag(pkg.version);
12
13  // तैनाती प्रक्रिया ट्रिगर करें
14  await triggerDeploy(pkg.version);
15}

उपयोग उदाहरण#

entry.node.ts
1import type { GezOptions } from '@gez/core';
2
3export default {
4  modules: {
5    // निर्यात करने के लिए आवश्यक मॉड्यूल कॉन्फ़िगर करें
6    exports: [
7      'root:src/components/button.vue',
8      'root:src/utils/format.ts',
9      'npm:vue',
10      'npm:vue-router'
11    ]
12  },
13  // पैकेजिंग कॉन्फ़िगरेशन
14  pack: {
15    // पैकेजिंग सुविधा सक्षम करें
16    enable: true,
17
18    // एक साथ कई संस्करण आउटपुट करें
19    outputs: [
20      'dist/versions/latest.tgz',
21      'dist/versions/1.0.0.tgz'
22    ],
23
24    // package.json को अनुकूलित करें
25    packageJson: async (gez, pkg) => {
26      pkg.version = '1.0.0';
27      return pkg;
28    },
29
30    // पैकेजिंग से पहले तैयारी
31    onBefore: async (gez, pkg) => {
32      // आवश्यक फ़ाइलें जोड़ें
33      await fs.writeFile('dist/README.md', '# Your App\n\nमॉड्यूल निर्यात स्पष्टीकरण...');
34      // टाइप जाँच निष्पादित करें
35      await runTypeCheck();
36    },
37
38    // पैकेजिंग के बाद प्रसंस्करण
39    onAfter: async (gez, pkg, file) => {
40      // निजी npm मिरर पर प्रकाशित करें
41      await publishToRegistry(file, {
42        registry: 'https://npm.your-registry.com/'
43      });
44      // या स्थिर सर्वर पर तैनात करें
45      await uploadToServer(file, 'https://static.example.com/packages');
46    }
47  }
48} satisfies GezOptions;
