Preact+HTM#

यह ट्यूटोरियल आपको Gez पर आधारित Preact+HTM SSR एप्लिकेशन को शुरू से बनाने में मदद करेगा। हम एक पूर्ण उदाहरण के माध्यम से दिखाएंगे कि Gez फ्रेमवर्क का उपयोग करके सर्वर-साइड रेंडरिंग एप्लिकेशन कैसे बनाया जाता है।

प्रोजेक्ट संरचना#

सबसे पहले, प्रोजेक्ट की मूल संरचना को समझते हैं:

1.
2├── package.json         # प्रोजेक्ट कॉन्फ़िगरेशन फ़ाइल, डिपेंडेंसी और स्क्रिप्ट कमांड को परिभाषित करती है
3├── tsconfig.json        # TypeScript कॉन्फ़िगरेशन फ़ाइल, कंपाइल विकल्प सेट करती है
4└── src                  # सोर्स कोड डायरेक्टरी
5    ├── app.ts           # मुख्य एप्लिकेशन कंपोनेंट, पेज संरचना और इंटरैक्शन लॉजिक को परिभाषित करता है
6    ├── create-app.ts    # एप्लिकेशन इंस्टेंस क्रिएशन फैक्टरी, एप्लिकेशन को इनिशियलाइज़ करने के लिए जिम्मेदार
7    ├── entry.client.ts  # क्लाइंट-साइड एंट्री फ़ाइल, ब्राउज़र-साइड रेंडरिंग को हैंडल करती है
8    ├── entry.node.ts    # Node.js सर्वर एंट्री फ़ाइल, डेवलपमेंट एनवायरनमेंट कॉन्फ़िगरेशन और सर्वर स्टार्टअप के लिए जिम्मेदार
9    └── entry.server.ts  # सर्वर-साइड एंट्री फ़ाइल, SSR रेंडरिंग लॉजिक को हैंडल करती है

package.json फ़ाइल बनाने के बाद, प्रोजेक्ट डिपेंडेंसी को इंस्टॉल करने की आवश्यकता है। आप निम्नलिखित किसी भी कमांड का उपयोग करके इंस्टॉल कर सकते हैं:

1pnpm install

सोर्स कोड संरचना#

app.ts#

मुख्य एप्लिकेशन कंपोनेंट src/app.ts बनाएं, Preact के क्लास कंपोनेंट और HTM का उपयोग करें:

src/app.ts
1/**
2 * @file उदाहरण कंपोनेंट
3 * @description Gez फ्रेमवर्क की मूल कार्यक्षमता को प्रदर्शित करने के लिए ऑटो-अपडेटिंग समय के साथ एक पेज टाइटल दिखाता है
4 */
5
6import { Component } from 'preact';
7import { html } from 'htm/preact';
8
9export default class App extends Component {
10    state = {
11        time: new Date().toISOString()
12    };
13
14    timer: NodeJS.Timeout | null = null;
15
16    componentDidMount() {
17        this.timer = setInterval(() => {
18            this.setState({
19                time: new Date().toISOString()
20            });
21        }, 1000);
22    }
23
24    componentWillUnmount() {
25        if (this.timer) {
26            clearInterval(this.timer);
27        }
28    }
29
30    render() {
31        const { time } = this.state;
32        return html`
33            <div>
34                <h1><a href="https://www.jsesm.com/guide/frameworks/preact-htm.html" target="_blank">Gez क्विक स्टार्ट</a></h1>
35                <time datetime=${time}>${time}</time>
36            </div>
37        `;
38    }
39}

create-app.ts#

src/create-app.ts फ़ाइल बनाएं, जो एप्लिकेशन इंस्टेंस बनाने के लिए जिम्मेदार है:

src/create-app.ts
1/**
2 * @file एप्लिकेशन इंस्टेंस क्रिएशन
3 * @description एप्लिकेशन इंस्टेंस को बनाने और कॉन्फ़िगर करने के लिए जिम्मेदार
4 */
5
6import type { VNode } from 'preact';
7import { html } from 'htm/preact';
8import App from './app';
9
10export function createApp(): { app: VNode } {
11    const app = html`<${App} />`;
12    return {
13        app
14    };
15}

src/entry.client.ts

entry.node.ts#

entry.node.ts फ़ाइल बनाएं, डेवलपमेंट एनवायरनमेंट और सर्वर स्टार्टअप को कॉन्फ़िगर करें:

src/entry.node.ts
1/**
2 * @file Node.js सर्वर एंट्री फ़ाइल
3 * @description डेवलपमेंट एनवायरनमेंट कॉन्फ़िगरेशन और सर्वर स्टार्टअप के लिए जिम्मेदार, SSR रनटाइम एनवायरनमेंट प्रदान करता है
4 */
5
6import http from 'node:http';
7import type { GezOptions } from '@gez/core';
8
9export default {
10    /**
11     * डेवलपमेंट एनवायरनमेंट के लिए एप्लिकेशन क्रिएटर को कॉन्फ़िगर करें
12     * @description Rspack एप्लिकेशन इंस्टेंस बनाएं और कॉन्फ़िगर करें, डेवलपमेंट एनवायरनमेंट के लिए बिल्ड और हॉट रीलोडिंग के लिए
13     * @param gez Gez फ्रेमवर्क इंस्टेंस, कोर फंक्शनलिटी और कॉन्फ़िगरेशन इंटरफेस प्रदान करता है
14     * @returns कॉन्फ़िगर किया गया Rspack एप्लिकेशन इंस्टेंस, HMR और रियल-टाइम प्रीव्यू को सपोर्ट करता है
15     */
16    async devApp(gez) {
17        return import('@gez/rspack').then((m) =>
18            m.createRspackHtmlApp(gez, {
19                config(context) {
20                    // यहां Rspack कंपाइल कॉन्फ़िगरेशन को कस्टमाइज़ करें
21                }
22            })
23        );
24    },
25
26    /**
27     * HTTP सर्वर को कॉन्फ़िगर और स्टार्ट करें
28     * @description HTTP सर्वर इंस्टेंस बनाएं, Gez मिडलवेयर को इंटीग्रेट करें, SSR रिक्वेस्ट को हैंडल करें
29     * @param gez Gez फ्रेमवर्क इंस्टेंस, मिडलवेयर और रेंडरिंग फंक्शनलिटी प्रदान करता है
30     */
31    async server(gez) {
32        const server = http.createServer((req, res) => {
33            // Gez मिडलवेयर का उपयोग करके रिक्वेस्ट को हैंडल करें
34            gez.middleware(req, res, async () => {
35                // सर्वर-साइड रेंडरिंग को एक्जीक्यूट करें
36                const rc = await gez.render({
37                    params: { url: req.url }
38                });
39                res.end(rc.html);
40            });
41        });
42
43        server.listen(3000, () => {
44            console.log('सर्वर शुरू: http://localhost:3000');
45        });
46    }
47} satisfies GezOptions;

यह फ़ाइल डेवलपमेंट एनवायरनमेंट कॉन्फ़िगरेशन और सर्वर स्टार्टअप के लिए एंट्री फ़ाइल है, जिसमें दो मुख्य कार्यक्षमताएं शामिल हैं:

  1. devApp फ़ंक्शन: डेवलपमेंट एनवायरनमेंट के लिए Rspack एप्लिकेशन इंस्टेंस को बनाने और कॉन्फ़िगर करने के लिए जिम्मेदार है, जो हॉट रीलोडिंग और रियल-टाइम प्रीव्यू को सपोर्ट करता है। यहां createRspackHtmlApp का उपयोग करके Preact+HTM के लिए विशेष Rspack एप्लिकेशन इंस्टेंस बनाया जाता है।
  2. server फ़ंक्शन: HTTP सर्वर को बनाने और कॉन्फ़िगर करने के लिए जिम्मेदार है, जो Gez मिडलवेयर को इंटीग्रेट करके SSR रिक्वेस्ट को हैंडल करता है।

entry.server.ts#

सर्वर-साइड रेंडरिंग एंट्री फ़ाइल src/entry.server.ts बनाएं:

src/entry.server.ts
1/**
2 * @file सर्वर-साइड रेंडरिंग एंट्री फ़ाइल
3 * @description सर्वर-साइड रेंडरिंग प्रोसेस, HTML जनरेशन और रिसोर्स इंजेक्शन के लिए जिम्मेदार
4 */
5
6import type { RenderContext } from '@gez/core';
7import type { VNode } from 'preact';
8import { render } from 'preact-render-to-string';
9import { createApp } from './create-app';
10
11export default async (rc: RenderContext) => {
12    // एप्लिकेशन इंस्टेंस बनाएं
13    const { app } = createApp();
14
15    // Preact के renderToString का उपयोग करके पेज कंटेंट जनरेट करें
16    const html = render(app);
17
18    // डिपेंडेंसी कलेक्शन को कमिट करें, यह सुनिश्चित करें कि सभी आवश्यक रिसोर्स लोड हो गए हैं
19    await rc.commit();
20
21    // पूर्ण HTML संरचना जनरेट करें
22    rc.html = `<!DOCTYPE html>
23<html lang="hi-IN">
24<head>
25    ${rc.preload()}
26    <title>Gez क्विक स्टार्ट</title>
27    ${rc.css()}
28</head>
29<body>
30    <div id="app">${html}</div>
31    ${rc.importmap()}
32    ${rc.moduleEntry()}
33    ${rc.modulePreload()}
34</body>
35</html>
36`;
37};

प्रोजेक्ट को रन करें#

उपरोक्त फ़ाइल कॉन्फ़िगरेशन को पूरा करने के बाद, आप निम्नलिखित कमांड का उपयोग करके प्रोजेक्ट को रन कर सकते हैं:

  1. डेवलपमेंट मोड:
1npm run dev
  1. प्रोजेक्ट बिल्ड करें:
1npm run build
  1. प्रोडक्शन एनवायरनमेंट में रन करें:
1npm run start

अब, आपने Gez पर आधारित Preact+HTM SSR एप्लिकेशन को सफलतापूर्वक बना लिया है! http://localhost:3000 पर जाकर परिणाम देखें।
