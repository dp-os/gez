# @gez/rspack-vue

Rspack Vue पैकेज Vue फ्रेमवर्क पर आधारित Rspack एप्लिकेशन बनाने और कॉन्फ़िगर करने के लिए एक API सेट प्रदान करता है, जो Vue कंपोनेंट डेवलपमेंट, बिल्ड और सर्वर-साइड रेंडरिंग का समर्थन करता है।

## इंस्टॉलेशन

पैकेज मैनेजर का उपयोग करके `@gez/rspack-vue` डेवलपमेंट डिपेंडेंसी इंस्टॉल करें:

<PackageManagerTabs command="install @gez/rspack-vue -D" />

## टाइप एक्सपोर्ट

### BuildTarget

```ts
type BuildTarget = 'node' | 'client' | 'server'
```

बिल्ड टार्गेट एनवायरनमेंट टाइप, जो एप्लिकेशन के बिल्ड टार्गेट एनवायरनमेंट को परिभाषित करता है, बिल्ड प्रक्रिया में विशिष्ट ऑप्टिमाइज़ेशन और फीचर्स को कॉन्फ़िगर करने के लिए उपयोग किया जाता है:
- `node`: Node.js एनवायरनमेंट में चलने वाले कोड के लिए बिल्ड करें
- `client`: ब्राउज़र एनवायरनमेंट में चलने वाले कोड के लिए बिल्ड करें
- `server`: सर्वर एनवायरनमेंट में चलने वाले कोड के लिए बिल्ड करें

### RspackAppConfigContext

```ts
interface RspackAppConfigContext {
  gez: Gez
  buildTarget: BuildTarget
  config: RspackOptions
  options: RspackAppOptions
}
```

Rspack एप्लिकेशन कॉन्फ़िगरेशन कॉन्टेक्स्ट इंटरफ़ेस, जो कॉन्फ़िगरेशन हुक फंक्शन में एक्सेस किए जा सकने वाले कॉन्टेक्स्ट जानकारी प्रदान करता है:
- `gez`: Gez फ्रेमवर्क इंस्टेंस
- `buildTarget`: वर्तमान बिल्ड टार्गेट (client/server/node)
- `config`: Rspack कॉन्फ़िगरेशन ऑब्जेक्ट
- `options`: एप्लिकेशन कॉन्फ़िगरेशन ऑप्शन्स

### RspackAppOptions

```ts
interface RspackAppOptions {
  css?: 'css' | 'style'
  loaders?: {
    styleLoader?: string
  }
  styleLoader?: Record<string, any>
  cssLoader?: Record<string, any>
  target?: {
    web?: string[]
    node?: string[]
  }
  definePlugin?: Record<string, any>
  config?: (context: RspackAppConfigContext) => void | Promise<void>
}
```

Rspack एप्लिकेशन कॉन्फ़िगरेशन ऑप्शन्स इंटरफ़ेस:
- `css`: CSS आउटपुट मोड, 'css' (अलग फाइल) या 'style' (इनलाइन स्टाइल) चुन सकते हैं
- `loaders`: कस्टम लोडर कॉन्फ़िगरेशन
- `styleLoader`: style-loader कॉन्फ़िगरेशन ऑप्शन्स
- `cssLoader`: css-loader कॉन्फ़िगरेशन ऑप्शन्स
- `target`: बिल्ड टार्गेट कंपैटिबिलिटी कॉन्फ़िगरेशन
- `definePlugin`: ग्लोबल कॉन्स्टेंट डेफिनिशन
- `config`: कॉन्फ़िगरेशन हुक फंक्शन

### RspackHtmlAppOptions

`RspackAppOptions` से इनहेरिट करता है, HTML एप्लिकेशन के लिए विशिष्ट ऑप्शन्स को कॉन्फ़िगर करने के लिए उपयोग किया जाता है।

## फंक्शन एक्सपोर्ट

### createRspackApp

```ts
function createRspackApp(gez: Gez, options?: RspackAppOptions): Promise<App>
```

एक स्टैंडर्ड Rspack एप्लिकेशन इंस्टेंस बनाएं।

**पैरामीटर्स:**
- `gez`: Gez फ्रेमवर्क इंस्टेंस
- `options`: Rspack एप्लिकेशन कॉन्फ़िगरेशन ऑप्शन्स

**रिटर्न वैल्यू:**
- एक Promise रिटर्न करता है, जो बनाए गए एप्लिकेशन इंस्टेंस को रिज़ॉल्व करता है

### createRspackHtmlApp

```ts
function createRspackHtmlApp(gez: Gez, options?: RspackHtmlAppOptions): Promise<App>
```

एक HTML टाइप का Rspack एप्लिकेशन इंस्टेंस बनाएं।

**पैरामीटर्स:**
- `gez`: Gez फ्रेमवर्क इंस्टेंस
- `options`: HTML एप्लिकेशन कॉन्फ़िगरेशन ऑप्शन्स

**रिटर्न वैल्यू:**
- एक Promise रिटर्न करता है, जो बनाए गए HTML एप्लिकेशन इंस्टेंस को रिज़ॉल्व करता है

## कॉन्स्टेंट एक्सपोर्ट
### RSPACK_LOADER

```ts
const RSPACK_LOADER: Record<string, string> = {
  builtinSwcLoader: 'builtin:swc-loader',
  lightningcssLoader: 'builtin:lightningcss-loader',
  styleLoader: 'style-loader',
  cssLoader: 'css-loader',
  lessLoader: 'less-loader',
  styleResourcesLoader: 'style-resources-loader',
  workerRspackLoader: 'worker-rspack-loader'
}
```

Rspack में बिल्ट-इन लोडर आइडेंटिफ़ायर मैपिंग ऑब्जेक्ट, जो सामान्यतः उपयोग किए जाने वाले लोडर नाम कॉन्स्टेंट प्रदान करता है:

- `builtinSwcLoader`: Rspack में बिल्ट-इन SWC लोडर, TypeScript/JavaScript फाइलों को प्रोसेस करने के लिए
- `lightningcssLoader`: Rspack में बिल्ट-इन lightningcss लोडर, CSS फाइलों के लिए हाई-परफॉर्मेंस कंपाइलर
- `styleLoader`: CSS को DOM में इंजेक्ट करने के लिए लोडर
- `cssLoader`: CSS फाइलों को पार्स करने और CSS मॉड्यूलरिटी को हैंडल करने के लिए लोडर
- `lessLoader`: Less फाइलों को CSS में कंपाइल करने के लिए लोडर
- `styleResourcesLoader`: ग्लोबल स्टाइल रिसोर्सेज (जैसे वेरिएबल्स, mixins) को ऑटो-इम्पोर्ट करने के लिए लोडर
- `workerRspackLoader`: Web Worker फाइलों को प्रोसेस करने के लिए लोडर

इन कॉन्स्टेंट्स का उपयोग करके कॉन्फ़िगरेशन में बिल्ट-इन लोडर्स को रेफरेंस किया जा सकता है, मैन्युअल स्ट्रिंग टाइपिंग से बचा जा सकता है:

```ts title="src/entry.node.ts"
import { RSPACK_LOADER } from '@gez/rspack';

export default {
  async devApp(gez) {
    return import('@gez/rspack').then((m) =>
      m.createRspackHtmlApp(gez, {
        loaders: {
          // कॉन्स्टेंट का उपयोग करके लोडर को रेफरेंस करें
          styleLoader: RSPACK_LOADER.styleLoader,
          cssLoader: RSPACK_LOADER.cssLoader,
          lightningcssLoader: RSPACK_LOADER.lightningcssLoader
        }
      })
    );
  }
};
```

**ध्यान दें:**
- ये लोडर्स Rspack में पहले से बिल्ट-इन हैं, अतिरिक्त इंस्टॉलेशन की आवश्यकता नहीं है
- कस्टम लोडर कॉन्फ़िगरेशन करते समय, इन कॉन्स्टेंट्स का उपयोग करके डिफ़ॉल्ट लोडर इम्प्लीमेंटेशन को रिप्लेस किया जा सकता है
- कुछ लोडर्स (जैसे `builtinSwcLoader`) के विशिष्ट कॉन्फ़िगरेशन ऑप्शन्स होते हैं, संबंधित कॉन्फ़िगरेशन डॉक्यूमेंटेशन का संदर्भ लें
