# त्वरित प्रारंभ

## प्रोजेक्ट इनिशियलाइज़ेशन

एक नया Gez प्रोजेक्ट बनाने और इनिशियलाइज़ करने के लिए, निम्नलिखित चरणों का पालन करें:

```bash
mkdir your-app-name
cd your-app-name
```

## फ्रेमवर्क संस्करण चयन

Gez फ्रेमवर्क कई तकनीकी स्टैक संस्करण प्रदान करता है, जिनमें से प्रत्येक को विभिन्न एप्लिकेशन परिदृश्यों के लिए अनुकूलित किया गया है। कृपया प्रोजेक्ट आवश्यकताओं के अनुसार उपयुक्त संस्करण चुनें:

### HTML

अत्यंत सरल आर्किटेक्चर वाले प्रोजेक्ट्स के लिए उपयुक्त:

- शून्य बाहरी निर्भरता, शुरू करने के लिए तैयार
- मूल JavaScript विकास अनुभव
- स्थिर वेबसाइट और हल्के एप्लिकेशन बनाने के लिए उपयुक्त
- प्रगतिशील कार्यक्षमता विस्तार का समर्थन

[HTML संस्करण का विस्तृत दस्तावेज़ देखें](../frameworks/html.md)

### Vue2

एंटरप्राइज़-ग्रेड एप्लिकेशन विकास के लिए उपयुक्त:

- पूर्ण TypeScript समर्थन
- समृद्ध तृतीय-पक्ष घटक पारिस्थितिकी तंत्र
- परिपक्व विकास टूलचेन
- उत्पादन वातावरण में सत्यापित स्थिरता

[Vue2 संस्करण का विस्तृत दस्तावेज़ देखें](../frameworks/vue2.md)

### Vue3

आधुनिक वेब एप्लिकेशन विकास के लिए उपयुक्त:

- Proxy आधारित प्रतिक्रियाशील प्रणाली
- कंपोज़ेबल API समर्थन
- बेहतर रनटाइम प्रदर्शन
- छोटा बंडल आकार

[Vue3 संस्करण का विस्तृत दस्तावेज़ देखें](../frameworks/vue3.md)

### Preact+HTM

हल्के और उच्च प्रदर्शन वाले प्रोजेक्ट्स के लिए उपयुक्त:

- अत्यंत छोटा रनटाइम आकार (3KB)
- मूल JavaScript टेम्पलेट सिंटैक्स
- React के साथ संगत API
- उत्कृष्ट प्रदर्शन

[Preact+HTM संस्करण का विस्तृत दस्तावेज़ देखें](../frameworks/preact-htm.md)

## AI सहायित विकास

Gez फ्रेमवर्क AI सहायित विकास क्षमता प्रदान करता है, जो विकास दक्षता को काफी बढ़ा सकता है:

1. उपयुक्त फ्रेमवर्क संस्करण दस्तावेज़ चुनें
2. दस्तावेज़ सामग्री को AI सहायक को प्रदान करें
3. AI स्वचालित रूप से प्रोजेक्ट स्केलेटन और कॉन्फ़िगरेशन फ़ाइलें उत्पन्न करेगा

:::tip सुझाव
AI सहायित विकास न केवल प्रोजेक्ट इनिशियलाइज़ेशन गति को बढ़ाता है, बल्कि यह सुनिश्चित करता है कि प्रोजेक्ट संरचना सर्वोत्तम प्रथाओं के अनुरूप हो।
:::
