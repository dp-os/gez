# @gez/rspack-vue

Rspack Vue 패키지는 Vue 프레임워크 기반의 Rspack 애플리케이션을 생성하고 구성하기 위한 API 세트를 제공하며, Vue 컴포넌트 개발, 빌드 및 서버 사이드 렌더링을 지원합니다.

## 설치

패키지 매니저를 사용하여 `@gez/rspack-vue` 개발 의존성을 설치합니다:

<PackageManagerTabs command="install @gez/rspack-vue -D" />

## 타입 내보내기

### BuildTarget

```ts
type BuildTarget = 'node' | 'client' | 'server'
```

빌드 대상 환경 타입으로, 애플리케이션의 빌드 대상 환경을 정의하며, 빌드 프로세스에서 특정 최적화 및 기능을 구성하는 데 사용됩니다:
- `node`: Node.js 환경에서 실행되는 코드로 빌드
- `client`: 브라우저 환경에서 실행되는 코드로 빌드
- `server`: 서버 환경에서 실행되는 코드로 빌드

### RspackAppConfigContext

```ts
interface RspackAppConfigContext {
  gez: Gez
  buildTarget: BuildTarget
  config: RspackOptions
  options: RspackAppOptions
}
```

Rspack 애플리케이션 구성 컨텍스트 인터페이스로, 구성 훅 함수에서 접근할 수 있는 컨텍스트 정보를 제공합니다:
- `gez`: Gez 프레임워크 인스턴스
- `buildTarget`: 현재 빌드 대상 (client/server/node)
- `config`: Rspack 구성 객체
- `options`: 애플리케이션 구성 옵션

### RspackAppOptions

```ts
interface RspackAppOptions {
  css?: 'css' | 'style'
  loaders?: {
    styleLoader?: string
  }
  styleLoader?: Record<string, any>
  cssLoader?: Record<string, any>
  target?: {
    web?: string[]
    node?: string[]
  }
  definePlugin?: Record<string, any>
  config?: (context: RspackAppConfigContext) => void | Promise<void>
}
```

Rspack 애플리케이션 구성 옵션 인터페이스:
- `css`: CSS 출력 방식, 'css' (독립 파일) 또는 'style' (인라인 스타일) 선택 가능
- `loaders`: 사용자 정의 loader 구성
- `styleLoader`: style-loader 구성 옵션
- `cssLoader`: css-loader 구성 옵션
- `target`: 빌드 대상 호환성 구성
- `definePlugin`: 전역 상수 정의
- `config`: 구성 훅 함수

### RspackHtmlAppOptions

`RspackAppOptions`를 상속받아 HTML 애플리케이션의 특정 옵션을 구성합니다.

## 함수 내보내기

### createRspackApp

```ts
function createRspackApp(gez: Gez, options?: RspackAppOptions): Promise<App>
```

표준 Rspack 애플리케이션 인스턴스를 생성합니다.

**매개변수:**
- `gez`: Gez 프레임워크 인스턴스
- `options`: Rspack 애플리케이션 구성 옵션

**반환값:**
- 생성된 애플리케이션 인스턴스로 해결되는 Promise 반환

### createRspackHtmlApp

```ts
function createRspackHtmlApp(gez: Gez, options?: RspackHtmlAppOptions): Promise<App>
```

HTML 타입의 Rspack 애플리케이션 인스턴스를 생성합니다.

**매개변수:**
- `gez`: Gez 프레임워크 인스턴스
- `options`: HTML 애플리케이션 구성 옵션

**반환값:**
- 생성된 HTML 애플리케이션 인스턴스로 해결되는 Promise 반환

## 상수 내보내기
### RSPACK_LOADER

```ts
const RSPACK_LOADER: Record<string, string> = {
  builtinSwcLoader: 'builtin:swc-loader',
  lightningcssLoader: 'builtin:lightningcss-loader',
  styleLoader: 'style-loader',
  cssLoader: 'css-loader',
  lessLoader: 'less-loader',
  styleResourcesLoader: 'style-resources-loader',
  workerRspackLoader: 'worker-rspack-loader'
}
```

Rspack 내장 loader 식별자 매핑 객체로, 일반적으로 사용되는 loader 이름 상수를 제공합니다:

- `builtinSwcLoader`: TypeScript/JavaScript 파일을 처리하기 위한 Rspack 내장 SWC loader
- `lightningcssLoader`: CSS 파일을 처리하기 위한 Rspack 내장 lightningcss loader
- `styleLoader`: CSS를 DOM에 주입하는 loader
- `cssLoader`: CSS 파일을 해석하고 CSS 모듈화를 처리하는 loader
- `lessLoader`: Less 파일을 CSS로 컴파일하는 loader
- `styleResourcesLoader`: 전역 스타일 리소스(예: 변수, mixins)를 자동으로 가져오는 loader
- `workerRspackLoader`: Web Worker 파일을 처리하는 loader

이러한 상수를 사용하여 구성에서 내장 loader를 참조할 수 있으며, 수동으로 문자열을 입력할 필요가 없습니다:

```ts title="src/entry.node.ts"
import { RSPACK_LOADER } from '@gez/rspack';

export default {
  async devApp(gez) {
    return import('@gez/rspack').then((m) =>
      m.createRspackHtmlApp(gez, {
        loaders: {
          // 상수를 사용하여 loader 참조
          styleLoader: RSPACK_LOADER.styleLoader,
          cssLoader: RSPACK_LOADER.cssLoader,
          lightningcssLoader: RSPACK_LOADER.lightningcssLoader
        }
      })
    );
  }
};
```

**주의사항:**
- 이러한 loader는 Rspack에 내장되어 있으므로 별도로 설치할 필요가 없습니다.
- 사용자 정의 loader 구성을 할 때, 이러한 상수를 사용하여 기본 loader 구현을 대체할 수 있습니다.
- 일부 loader(예: `builtinSwcLoader`)는 특정 구성 옵션이 있으므로 해당 구성 문서를 참조하십시오.
