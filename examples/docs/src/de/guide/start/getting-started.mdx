# Schnellstart

## Projektinitialisierung

Um ein neues Gez-Projekt zu erstellen und zu initialisieren, führen Sie die folgenden Schritte aus:

```bash
mkdir your-app-name
cd your-app-name
```

## Framework-Version-Auswahl

Das Gez Framework bietet mehrere Technologie-Stack-Versionen, die jeweils für verschiedene Anwendungsszenarien optimiert sind. Wählen Sie die passende Version basierend auf Ihren Projektanforderungen:

### HTML

Geeignet für Projekte, die eine minimalistisches Architektur bevorzugen:

- Keine externen Abhängigkeiten, sofort einsatzbereit
- Native JavaScript-Entwicklungserfahrung
- Ideal für statische Websites und leichtgewichtige Anwendungen
- Unterstützung für progressive Funktionserweiterungen

[HTML-Version Dokumentation anzeigen](../frameworks/html.md)

### Vue2

Geeignet für die Entwicklung von Unternehmensanwendungen:

- Vollständige TypeScript-Unterstützung
- Umfangreiches Ökosystem von Drittanbieter-Komponenten
- Ausgereifte Entwicklungstoolchain
- In Produktionsumgebungen bewährte Stabilität

[Vue2-Version Dokumentation anzeigen](../frameworks/vue2.md)

### Vue3

Geeignet für die Entwicklung moderner Webanwendungen:

- Proxy-basiertes reaktives System
- Unterstützung für Composition API
- Bessere Laufzeitleistung
- Kleinere Bundle-Größe

[Vue3-Version Dokumentation anzeigen](../frameworks/vue3.md)

### Preact+HTM

Geeignet für Projekte, die Leichtgewichtigkeit und hohe Leistung bevorzugen:

- Sehr kleine Laufzeitgröße (3KB)
- Native JavaScript-Templatesyntax
- Mit React kompatible API
- Hervorragende Leistung

[Preact+HTM-Version Dokumentation anzeigen](../frameworks/preact-htm.md)

## KI-unterstützte Entwicklung

Das Gez Framework bietet KI-unterstützte Entwicklungsfähigkeiten, die die Entwicklungseffizienz erheblich steigern können:

1. Wählen Sie die passende Framework-Version aus der Dokumentation
2. Geben Sie den Dokumentationsinhalt an den KI-Assistenten weiter
3. Der KI-Assistent generiert automatisch das Projektskelett und die Konfigurationsdateien

:::tip Tipp
Die KI-unterstützte Entwicklung beschleunigt nicht nur die Projektinitialisierung, sondern stellt auch sicher, dass die Projektstruktur den Best Practices entspricht.
:::
