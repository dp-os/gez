# Quick Start

## Project Initialization

To create and initialize a new Gez project, follow these steps:

```bash
mkdir your-app-name
cd your-app-name
```

## Framework Version Selection

The Gez Framework offers multiple technology stack versions, each optimized for different application scenarios. Choose the appropriate version based on your project requirements:

### HTML

Suitable for projects pursuing a minimalist architecture:

- Zero external dependencies, ready to use out of the box
- Native JavaScript development experience
- Ideal for building static websites and lightweight applications
- Supports progressive feature expansion

[View HTML Version Detailed Documentation](../frameworks/html.md)

### Vue2

Suitable for enterprise-level application development:

- Full TypeScript support
- Rich third-party component ecosystem
- Mature development toolchain
- Production-proven stability

[View Vue2 Version Detailed Documentation](../frameworks/vue2.md)

### Vue3

Suitable for modern web application development:

- Proxy-based reactivity system
- Composition API support
- Improved runtime performance
- Smaller bundle size

[View Vue3 Version Detailed Documentation](../frameworks/vue3.md)

### Preact+HTM

Suitable for projects pursuing lightweight and high performance:

- Extremely small runtime size (3KB)
- Native JavaScript template syntax
- React-compatible API
- Excellent performance

[View Preact+HTM Version Detailed Documentation](../frameworks/preact-htm.md)

## AI-Assisted Development

The Gez Framework provides AI-assisted development capabilities, significantly improving development efficiency:

1. Select the appropriate framework version documentation
2. Provide the documentation content to the AI assistant
3. The AI will automatically generate the project skeleton and configuration files

:::tip Tip
AI-assisted development not only speeds up project initialization but also ensures the project structure adheres to best practices.
:::
