# @gez/rspack

Rspack 套件提供了一套用於建立和配置 Rspack 應用的 API，支援標準應用和 HTML 應用的建置與開發。

## 安裝

使用套件管理器安裝 `@gez/rspack` 開發依賴：

<PackageManagerTabs command="install @gez/rspack -D" />

## 類型匯出

### BuildTarget

```ts
type BuildTarget = 'node' | 'client' | 'server'
```

建置目標環境類型，定義了應用程式的建置目標環境，用於配置建置過程中的特定優化和功能：
- `node`: 建置為 Node.js 環境執行的程式碼
- `client`: 建置為瀏覽器環境執行的程式碼
- `server`: 建置為伺服器環境執行的程式碼

### RspackAppConfigContext

```ts
interface RspackAppConfigContext {
  gez: Gez
  buildTarget: BuildTarget
  config: RspackOptions
  options: RspackAppOptions
}
```

Rspack 應用配置上下文介面，提供了在配置鉤子函式中可以存取的上下文資訊：
- `gez`: Gez 框架實例
- `buildTarget`: 當前的建置目標（client/server/node）
- `config`: Rspack 配置物件
- `options`: 應用配置選項

### RspackAppOptions

```ts
interface RspackAppOptions {
  css?: 'css' | 'style'
  loaders?: {
    styleLoader?: string
  }
  styleLoader?: Record<string, any>
  cssLoader?: Record<string, any>
  target?: {
    web?: string[]
    node?: string[]
  }
  definePlugin?: Record<string, any>
  config?: (context: RspackAppConfigContext) => void | Promise<void>
}
```

Rspack 應用配置選項介面：
- `css`: CSS 輸出方式，可選 'css'（獨立檔案）或 'style'（內嵌樣式）
- `loaders`: 自訂 loader 配置
- `styleLoader`: style-loader 配置選項
- `cssLoader`: css-loader 配置選項
- `target`: 建置目標相容性配置
- `definePlugin`: 全域常數定義
- `config`: 配置鉤子函式

### RspackHtmlAppOptions

繼承自 `RspackAppOptions`，用於配置 HTML 應用的特定選項。

## 函式匯出

### createRspackApp

```ts
function createRspackApp(gez: Gez, options?: RspackAppOptions): Promise<App>
```

建立一個標準的 Rspack 應用實例。

**參數：**
- `gez`: Gez 框架實例
- `options`: Rspack 應用配置選項

**回傳值：**
- 回傳一個 Promise，解析為建立的應用實例

### createRspackHtmlApp

```ts
function createRspackHtmlApp(gez: Gez, options?: RspackHtmlAppOptions): Promise<App>
```

建立一個 HTML 類型的 Rspack 應用實例。

**參數：**
- `gez`: Gez 框架實例
- `options`: HTML 應用配置選項

**回傳值：**
- 回傳一個 Promise，解析為建立的 HTML 應用實例

## 常數匯出
### RSPACK_LOADER

```ts
const RSPACK_LOADER: Record<string, string> = {
  builtinSwcLoader: 'builtin:swc-loader',
  lightningcssLoader: 'builtin:lightningcss-loader',
  styleLoader: 'style-loader',
  cssLoader: 'css-loader',
  lessLoader: 'less-loader',
  styleResourcesLoader: 'style-resources-loader',
  workerRspackLoader: 'worker-rspack-loader'
}
```

Rspack 內建的 loader 識別符映射物件，提供了常用的 loader 名稱常數：

- `builtinSwcLoader`: Rspack 內建的 SWC loader，用於處理 TypeScript/JavaScript 檔案
- `lightningcssLoader`: Rspack 內建的 lightningcss loader，用於處理 CSS 檔案的高效能編譯器
- `styleLoader`: 用於將 CSS 注入到 DOM 中的 loader
- `cssLoader`: 用於解析 CSS 檔案和處理 CSS 模組化的 loader
- `lessLoader`: 用於將 Less 檔案編譯為 CSS 的 loader
- `styleResourcesLoader`: 用於自動匯入全域樣式資源（如變數、mixins）的 loader
- `workerRspackLoader`: 用於處理 Web Worker 檔案的 loader

使用這些常數可以在配置中引用內建的 loader，避免手動輸入字串：

```ts title="src/entry.node.ts"
import { RSPACK_LOADER } from '@gez/rspack';

export default {
  async devApp(gez) {
    return import('@gez/rspack').then((m) =>
      m.createRspackHtmlApp(gez, {
        loaders: {
          // 使用常數引用 loader
          styleLoader: RSPACK_LOADER.styleLoader,
          cssLoader: RSPACK_LOADER.cssLoader,
          lightningcssLoader: RSPACK_LOADER.lightningcssLoader
        }
      })
    );
  }
};
```

**注意事項：**
- 這些 loader 已經內建在 Rspack 中，無需額外安裝
- 在自訂 loader 配置時，可以使用這些常數來替換預設的 loader 實作
- 某些 loader（如 `builtinSwcLoader`）有特定的配置選項，請參考相應的配置文件
