Preact+HTM#

Ce tutoriel vous guidera dans la création d'une application SSR Preact+HTM basée sur Gez à partir de zéro. Nous allons utiliser un exemple complet pour montrer comment utiliser le framework Gez pour créer une application avec rendu côté serveur.

Structure du projet#

Commençons par comprendre la structure de base du projet :

1.
2├── package.json         # Fichier de configuration du projet, définissant les dépendances et les scripts
3├── tsconfig.json        # Fichier de configuration TypeScript, définissant les options de compilation
4└── src                  # Répertoire des sources
5    ├── app.ts           # Composant principal de l'application, définissant la structure de la page et la logique d'interaction
6    ├── create-app.ts    # Factory de création d'instance d'application, responsable de l'initialisation de l'application
7    ├── entry.client.ts  # Fichier d'entrée côté client, gérant le rendu côté navigateur
8    ├── entry.node.ts    # Fichier d'entrée Node.js, responsable de la configuration de l'environnement de développement et du démarrage du serveur
9    └── entry.server.ts  # Fichier d'entrée côté serveur, gérant la logique de rendu SSR

Configuration du projet#

package.json#

Créez le fichier package.json pour configurer les dépendances et les scripts du projet :

package.json
1{
2  "name": "ssr-demo-preact-htm",
3  "version": "1.0.0",
4  "type": "module",
5  "private": true,
6  "scripts": {
7    "dev": "gez dev",
8    "build": "npm run build:dts && npm run build:ssr",
9    "build:ssr": "gez build",
10    "preview": "gez preview",
11    "start": "NODE_ENV=production node dist/index.js",
12    "build:dts": "tsc --declaration --emitDeclarationOnly --outDir dist/src"
13  },
14  "dependencies": {
15    "@gez/core": "*"
16  },
17  "devDependencies": {
18    "@gez/rspack": "*",
19    "@types/node": "22.8.6",
20    "htm": "^3.1.1",
21    "preact": "^10.26.2",
22    "preact-render-to-string": "^6.5.13",
23    "typescript": "^5.2.2"
24  }
25}

Après avoir créé le fichier package.json, vous devez installer les dépendances du projet. Vous pouvez utiliser l'une des commandes suivantes pour l'installation :

1pnpm install
2# ou
3yarn install
4# ou
5npm install

Cela installera tous les packages nécessaires, y compris Preact, HTM, TypeScript et les dépendances liées au SSR.

Structure du code source#

app.ts#

Créez le composant principal de l'application src/app.ts, en utilisant les composants de classe de Preact et HTM :

src/app.ts
1/**
2 * @file Composant d'exemple
3 * @description Affiche un titre de page avec une mise à jour automatique de l'heure, pour démontrer les fonctionnalités de base du framework Gez
4 */
5
6import { Component } from 'preact';
7import { html } from 'htm/preact';
8
9export default class App extends Component {
10    state = {
11        time: new Date().toISOString()
12    };
13
14    timer: NodeJS.Timeout | null = null;
15
16    componentDidMount() {
17        this.timer = setInterval(() => {
18            this.setState({
19                time: new Date().toISOString()
20            });
21        }, 1000);
22    }
23
24    componentWillUnmount() {
25        if (this.timer) {
26            clearInterval(this.timer);
27        }
28    }
29
30    render() {
31        const { time } = this.state;
32        return html`
33            <div>
34                <h1><a href="https://www.jsesm.com/guide/frameworks/preact-htm.html" target="_blank">Démarrage rapide avec Gez</a></h1>
35                <time datetime=${time}>${time}</time>
36            </div>
37        `;
38    }
39}

create-app.ts#

Créez le fichier src/create-app.ts, responsable de la création de l'instance de l'application :

src/create-app.ts
1/**
2 * @file Création de l'instance de l'application
3 * @description Responsable de la création et de la configuration de l'instance de l'application
4 */
5
6import type { VNode } from 'preact';
7import { html } from 'htm/preact';
8import App from './app';
9
10export function createApp(): { app: VNode } {
11    const app = html`<${App} />`;
12    return {
13        app
14    };
15}

Créez le fichier d'entrée côté client src/entry.client.ts :

src/entry.client.ts

4 */

6import { render } from 'preact';
7import { createApp } from './create-app';
8
9// Créez l'instance de l'application
10const { app } = createApp();
11
12// Montez l'instance de l'application
13render(app, document.getElementById('app')!);

entry.node.ts#

Créez le fichier entry.node.ts, configurant l'environnement de développement et le démarrage du serveur :

src/entry.node.ts
1/**
2 * @file Fichier d'entrée Node.js
3 * @description Responsable de la configuration de l'environnement de développement et du démarrage du serveur, fournissant l'environnement d'exécution SSR
4 */
5
6import http from 'node:http';
7import type { GezOptions } from '@gez/core';
8
9export default {
10    /**
11     * Configure le créateur d'application pour l'environnement de développement
12     * @description Crée et configure l'instance d'application Rspack pour la construction et la mise à jour en temps réel
13     * @param gez Instance du framework Gez, fournissant les fonctionnalités de base et les interfaces de configuration
14     * @returns Retourne l'instance d'application Rspack configurée, prenant en charge HMR et la prévisualisation en temps réel
15     */
16    async devApp(gez) {
17        return import('@gez/rspack').then((m) =>
18            m.createRspackHtmlApp(gez, {
19                config(context) {
20                    // Personnalisez ici la configuration de compilation Rspack
21                }
22            })
23        );
24    },
25
26    /**
27     * Configure et démarre le serveur HTTP
28     * @description Crée une instance de serveur HTTP, intègre le middleware Gez, et gère les requêtes SSR
29     * @param gez Instance du framework Gez, fournissant le middleware et les fonctionnalités de rendu
30     */
31    async server(gez) {
32        const server = http.createServer((req, res) => {
33            // Utilisez le middleware Gez pour traiter les requêtes
34            gez.middleware(req, res, async () => {
35                // Exécutez le rendu côté serveur
36                const rc = await gez.render({
37                    params: { url: req.url }
38                });
39                res.end(rc.html);
40            });
41        });
42
43        server.listen(3000, () => {
44            console.log('Serveur démarré : http://localhost:3000');
45        });
46    }
47} satisfies GezOptions;

Ce fichier est le point d'entrée pour la configuration de l'environnement de développement et le démarrage du serveur. Il contient deux fonctionnalités principales :

  1. La fonction devApp : responsable de la création et de la configuration de l'instance d'application Rspack pour l'environnement de développement, prenant en charge la mise à jour en temps réel et la prévisualisation. Ici, createRspackHtmlApp est utilisé pour créer une instance d'application Rspack spécifique à Preact+HTM.
  2. La fonction server : responsable de la création et de la configuration du serveur HTTP, intégrant le middleware Gez pour traiter les requêtes SSR.

entry.server.ts#

Créez le fichier d'entrée pour le rendu côté serveur src/entry.server.ts :

src/entry.server.ts
1/**
2 * @file Fichier d'entrée pour le rendu côté serveur
3 * @description Responsable du processus de rendu côté serveur, de la génération du HTML et de l'injection des ressources
4 */
5
6import type { RenderContext } from '@gez/core';
7import type { VNode } from 'preact';
8import { render } from 'preact-render-to-string';
9import { createApp } from './create-app';
10
11export default async (rc: RenderContext) => {
12    // Créez l'instance de l'application
13    const { app } = createApp();
14
15    // Utilisez renderToString de Preact pour générer le contenu de la page
16    const html = render(app);
17
18    // Soumettez la collecte des dépendances pour garantir que toutes les ressources nécessaires sont chargées
19    await rc.commit();
20
21    // Générez la structure HTML complète
22    rc.html = `<!DOCTYPE html>
23<html lang="fr">
24<head>
25    ${rc.preload()}
26    <title>Démarrage rapide avec Gez</title>
27    ${rc.css()}
28</head>
29<body>
30    <div id="app">${html}</div>
31    ${rc.importmap()}
32    ${rc.moduleEntry()}
33    ${rc.modulePreload()}
34</body>
35</html>
36`;
37};

Exécution du projet#

Après avoir configuré les fichiers ci-dessus, vous pouvez utiliser les commandes suivantes pour exécuter le projet :

  1. Mode développement :
1npm run dev
  1. Construction du projet :
1npm run build
  1. Exécution en production :
1npm run start

Vous avez maintenant créé avec succès une application SSR Preact+HTM basée sur Gez ! Visitez http://localhost:3000 pour voir le résultat.
