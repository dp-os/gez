RenderContext#

RenderContext là lớp cốt lõi trong framework Gez, chịu trách nhiệm quản lý vòng đời hoàn chỉnh của render phía máy chủ (SSR). Nó cung cấp một bộ API hoàn chỉnh để xử lý các tác vụ chính như render context, quản lý tài nguyên, đồng bộ trạng thái:

  • Điều khiển render: Quản lý quy trình render phía máy chủ, hỗ trợ các tình huống như render nhiều entry, render có điều kiện
  • Quản lý tài nguyên: Thu thập và chèn thông minh các tài nguyên tĩnh như JS, CSS để tối ưu hiệu suất tải
  • Đồng bộ trạng thái: Xử lý tuần tự hóa trạng thái phía máy chủ, đảm bảo client kích hoạt chính xác (hydration)
  • Điều khiển định tuyến: Hỗ trợ các tính năng nâng cao như chuyển hướng phía máy chủ, thiết lập mã trạng thái

Định nghĩa kiểu#

ServerRenderHandle#

Định nghĩa kiểu cho hàm xử lý render phía máy chủ.

1type ServerRenderHandle = (rc: RenderContext) => Promise<void> | void;

Hàm xử lý render phía máy chủ là một hàm đồng bộ hoặc bất đồng bộ, nhận một thể hiện của RenderContext làm tham số, dùng để xử lý logic render phía máy chủ.

entry.node.ts
1// 1. Hàm xử lý bất đồng bộ
2export default async (rc: RenderContext) => {
3  const app = createApp();
4  const html = await renderToString(app);
5  rc.html = html;
6};
7
8// 2. Hàm xử lý đồng bộ
9export const simple = (rc: RenderContext) => {
10  rc.html = '<h1>Hello World</h1>';
11};

RenderFiles#

Định nghĩa kiểu cho danh sách các tệp tài nguyên được thu thập trong quá trình render.

1interface RenderFiles {
2  js: string[];
3  css: string[];
4  modulepreload: string[];
5  resources: string[];
6}
  • js: Danh sách các tệp JavaScript
  • css: Danh sách các tệp stylesheet
  • modulepreload: Danh sách các module ESM cần preload
  • resources: Danh sách các tệp tài nguyên khác (hình ảnh, font, v.v.)
1// Ví dụ danh sách tệp tài nguyên
2rc.files = {
3  js: [
4    '/assets/entry-client.js',
5    '/assets/vendor.js'
6  ],
7  css: [
8    '/assets/main.css',
9    '/assets/vendor.css'
10  ],
11  modulepreload: [
12    '/assets/Home.js',
13    '/assets/About.js'
14  ],
15  resources: [
16    '/assets/logo.png',
17    '/assets/font.woff2'
18  ]
19};

ImportmapMode#

Định nghĩa chế độ tạo importmap.

1type ImportmapMode = 'inline' | 'js';
  • inline: Nội dung importmap được nhúng trực tiếp vào HTML, phù hợp với các tình huống:
    • Cần giảm số lượng yêu cầu HTTP
    • Nội dung importmap nhỏ
    • Yêu cầu hiệu suất tải trang đầu tiên cao
  • js: Nội dung importmap được tạo thành một tệp JS độc lập, phù hợp với các tình huống:
    • Nội dung importmap lớn
    • Cần tận dụng cơ chế cache của trình duyệt
    • Nhiều trang chia sẻ cùng một importmap

Lớp render context, chịu trách nhiệm quản lý tài nguyên và tạo HTML trong quá trình render phía máy chủ (SSR).

Tùy chọn thể hiện#

Định nghĩa các tùy chọn cấu hình cho render context.

1interface RenderContextOptions {
2  base?: string
3  entryName?: string
4  params?: Record<string, any>
5  importmapMode?: ImportmapMode
6}

base#

  • Kiểu: string
  • Mặc định: ''

Đường dẫn cơ sở cho các tài nguyên tĩnh.

  • Tất cả các tài nguyên tĩnh (JS, CSS, hình ảnh, v.v.) sẽ được tải dựa trên đường dẫn này
  • Hỗ trợ cấu hình động trong thời gian chạy, không cần build lại
  • Thường được sử dụng trong các tình huống như trang web đa ngôn ngữ, ứng dụng micro frontend

entryName#

  • Kiểu: string
  • Mặc định: 'default'

Tên hàm entry render phía máy chủ. Dùng để chỉ định hàm entry được sử dụng khi render phía máy chủ, khi một module xuất ra nhiều hàm render.

src/entry.server.ts
1export const mobile = async (rc: RenderContext) => {
2  // Logic render cho thiết bị di động
3};
4
5export const desktop = async (rc: RenderContext) => {
6  // Logic render cho máy tính để bàn
7};

params#

  • Kiểu: Record<string, any>
  • Mặc định: {}

Các tham số render. Có thể truyền bất kỳ loại tham số nào cho hàm render, thường được sử dụng để truyền thông tin yêu cầu (URL, tham số query, v.v.).

1const rc = await gez.render({
2  params: {
3    url: req.url,
4    lang: 'zh-CN',
5    theme: 'dark'
6  }
7});

importmapMode#

  • Kiểu: 'inline' | 'js'
  • Mặc định: 'inline'

Chế độ tạo import map:

  • inline: Nội dung importmap được nhúng trực tiếp vào HTML

  • js: Nội dung importmap được tạo thành một tệp JS độc lập

Thuộc tính thể hiện#

gez#

  • Kiểu: Gez
  • Chỉ đọc: true

Tham chiếu đến thể hiện Gez. Dùng để truy cập các chức năng và thông tin cấu hình cốt lõi của framework.

redirect#

  • Kiểu: string | null
  • Mặc định: null

Địa chỉ chuyển hướng. Khi được thiết lập, máy chủ có thể thực hiện chuyển hướng HTTP dựa trên giá trị này, thường được sử dụng trong các tình huống như xác thực đăng nhập, kiểm soát quyền truy cập.

entry.node.ts
1// Ví dụ xác thực đăng nhập
2export default async (rc: RenderContext) => {
3  if (!isLoggedIn()) {
4    rc.redirect = '/login';
5    rc.status = 302;
6    return;
7  }
8  // Tiếp tục render trang...
9};
10
11// Ví dụ kiểm soát quyền truy cập
12export default async (rc: RenderContext) => {
13  if (!hasPermission()) {
14    rc.redirect = '/403';
15    rc.status = 403;
16    return;
17  }
18  // Tiếp tục render trang...
19};

status#

  • Kiểu: number | null
  • Mặc định: null

Mã trạng thái HTTP phản hồi. Có thể thiết lập bất kỳ mã trạng thái HTTP hợp lệ nào, thường được sử dụng trong các tình huống xử lý lỗi, chuyển hướng.

entry.node.ts
1// Ví dụ xử lý lỗi 404
2export default async (rc: RenderContext) => {
3  const page = await findPage(rc.params.url);
4  if (!page) {
5    rc.status = 404;
6    // Render trang 404...
7    return;
8  }
9  // Tiếp tục render trang...
10};
11
12// Ví dụ chuyển hướng tạm thời
13export default async (rc: RenderContext) => {
14  if (needMaintenance()) {
15    rc.redirect = '/maintenance';
16    rc.status = 307; // Chuyển hướng tạm thời, giữ nguyên phương thức yêu cầu
17    return;
18  }
19  // Tiếp tục render trang...
20};

html#

  • Kiểu: string
  • Mặc định: ''

Nội dung HTML. Dùng để thiết lập và lấy nội dung HTML cuối cùng được tạo ra, khi thiết lập sẽ tự động xử lý các placeholder đường dẫn cơ sở.

entry.node.ts
1// Cách sử dụng cơ bản
2export default async (rc: RenderContext) => {
3  // Thiết lập nội dung HTML
4  rc.html = `
5    <!DOCTYPE html>
6    <html>
7      <head>
8        ${rc.preload()}
9        ${rc.css()}
10      </head>
11      <body>
12        <div id="app">Hello World</div>
13        ${rc.importmap()}
14        ${rc.moduleEntry()}
15        ${rc.modulePreload()}
16      </body>
17    </html>
18  `;
19};
20
21// Đường dẫn cơ sở động
22const rc = await gez.render({
23  base: '/app',  // Thiết lập đường dẫn cơ sở
24  params: { url: req.url }
25});
26
27// Các placeholder trong HTML sẽ được thay thế tự động:
28// [[[___GEZ_DYNAMIC_BASE___]]]/your-app-name/css/style.css
29// Thay thế thành:
30// /app/your-app-name/css/style.css

base#

  • Kiểu: string
  • Chỉ đọc: true
  • Mặc định: ''

Đường dẫn cơ sở cho các tài nguyên tĩnh. Tất cả các tài nguyên tĩnh (JS, CSS, hình ảnh, v.v.) sẽ được tải dựa trên đường dẫn này, hỗ trợ cấu hình động trong thời gian chạy.

1// Cách sử dụng cơ bản
2const rc = await gez.render({
3  base: '/gez',  // Thiết lập đường dẫn cơ sở
4  params: { url: req.url }
5});
6
7// Ví dụ trang web đa ngôn ngữ
8const rc = await gez.render({
9  base: '/cn',  // Trang web tiếng Trung
10  params: { lang: 'zh-CN' }
11});
12
13// Ví dụ ứng dụng micro frontend
14const rc = await gez.render({
15  base: '/app1',  // Ứng dụng con 1
16  params: { appId: 1 }
17});

entryName#

  • Kiểu: string
  • Chỉ đọc: true
  • Mặc định: 'default'

Tên hàm entry render phía máy chủ. Dùng để chọn hàm render từ entry.server.ts.

entry.node.ts
1// Hàm entry mặc định
2export default async (rc: RenderContext) => {
3  // Logic render mặc định
4};
5
6// Nhiều hàm entry
7export const mobile = async (rc: RenderContext) => {
8  // Logic render cho thiết bị di động
9};
10
11export const desktop = async (rc: RenderContext) => {
12  // Logic render cho máy tính để bàn
13};
14
15// Chọn hàm entry dựa trên loại thiết bị
16const rc = await gez.render({
17  entryName: isMobile ? 'mobile' : 'desktop',
18  params: { url: req.url }
19});

params#

  • Kiểu: Record<string, any>
  • Chỉ đọc: true
  • Mặc định: {}

Các tham số render. Có thể truyền và truy cập các tham số trong quá trình render phía máy chủ, thường được sử dụng để truyền thông tin yêu cầu, cấu hình trang.

1// Cách sử dụng cơ bản - Truyền URL và cài đặt ngôn ngữ
2const rc = await gez.render({
3  params: {
4    url: req.url,
5    lang: 'zh-CN'
6  }
7});
8
9// Cấu hình trang - Thiết lập chủ đề và bố cục
10const rc = await gez.render({
11  params: {
12    theme: 'dark',
13    layout: 'sidebar'
14  }
15});
16
17// Cấu hình môi trường - Chèn địa chỉ API
18const rc = await gez.render({
19  params: {
20    apiBaseUrl: process.env.API_BASE_URL,
21    version: '1.0.0'
22  }
23});

importMetaSet#

  • Kiểu: Set<ImportMeta>

Tập hợp thu thập phụ thuộc module. Tự động theo dõi và ghi lại các phụ thuộc module trong quá trình render component, chỉ thu thập các tài nguyên thực sự được sử dụng khi render trang hiện tại.

1// Cách sử dụng cơ bản
2const renderToString = (app: any, context: { importMetaSet: Set<ImportMeta> }) => {
3  // Tự động thu thập phụ thuộc module trong quá trình render
4  // Framework sẽ tự động gọi context.importMetaSet.add(import.meta) khi render component
5  // Nhà phát triển không cần xử lý thu thập phụ thuộc thủ công
6  return '<div id="app">Hello World</div>';
7};
8
9// Ví dụ sử dụng
10const app = createApp();
11const html = await renderToString(app, {
12  importMetaSet: rc.importMetaSet
13});

  • modulepreload: Danh sách các module ESM cần preload
  • resources: Danh sách các tệp tài nguyên khác (hình ảnh, font, v.v.)
1// Thu thập tài nguyên
2await rc.commit();
3
4// Chèn tài nguyên
5rc.html = `
6  <!DOCTYPE html>
7  <html>

19  </body>
20  </html>
21`;

Phương thức thể hiện#

serialize()#

  • Tham số:
    • input: any - Dữ liệu cần tuần tự hóa
    • options?: serialize.SerializeJSOptions - Tùy chọn tuần tự hóa
  • Trả về: string

Tuần tự hóa đối tượng JavaScript thành chuỗi. Dùng để tuần tự hóa dữ liệu trạng thái trong quá trình render phía máy chủ, đảm bảo dữ liệu có thể được nhúng an toàn vào HTML.

1const state = {
2  user: { id: 1, name: 'Alice' },
3  timestamp: new Date()
4};
5
6rc.html = `
7  <script>
8    window.__INITIAL_STATE__ = ${rc.serialize(state)};
9  </script>
10`;

state()#

  • Tham số:
    • varName: string - Tên biến
